Алгоритм:
1. Получаю объект дерева html страницы.
2. Перебираю элементы дерева.
3. Для каждого элемента, получаю суммарный текст вложенных элементов, глубина вложенности задается параметрами.
4. Суммарный текст элемента сохраняется, для последующего выбора наиболее оптимального.
5. Самый простой способ, выбрать самый длинный суммарный текст элемента при глубине вложенности 2.

Результаты:
1. Список протестированы страниц представлен в default.config.ini в [URLS] [TEST_URLS].
2. Вывод сохранен в каталоге test_pages_20160621_0813/ .

Улучшения:
1. Реализовать более сложный фильтр по допустимым и не допустимым тэгам при формировании текста элемента.
2. Усложнить алгоритм отбора оптимального суммарного текст элемента для большей глубины вложенности.
3. Использовать веб движок для полного анализа отображаемой информации (верстки, стилей и т.д.).
